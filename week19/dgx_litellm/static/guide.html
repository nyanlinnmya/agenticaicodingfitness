💻 vs 🖥️ — where do I run each command?

💻 Your laptop runs this tutorial (the web app you're looking at). 🖥️ The DGX runs the real models + the LiteLLM gateway. The laptop talks to the DGX over the connection shown in the mode pill (top-right).

The golden rule

  • 💻 The … tutorial_server.py command (how you launched this) and any export DGX_… run on your laptop.
  • 🖥️ Every grey shell block inside a chapter (litellm --config, docker, ollama, vllm, ngrok…) runs on the DGX. The app only shows them — it never runs them on your laptop.

Connect this app to your DGX backend (set on the 💻 laptop, then reload)

# ① backend on your LAN (or this laptop)
export DGX_CONN=local

# ② backend on a DGX in another network, over an ngrok tunnel
export DGX_CONN=tunnel
export DGX_TUNNEL_URL="https://user:pass@<id>.ngrok-free.app/v1"
#   🖥️ on the DGX:  ngrok http 11434 --host-header="localhost:11434" --basic-auth="user:pass"

# ③ a cloud provider (Ollama Cloud, etc.)
export DGX_CONN=cloud
export DGX_CLOUD_URL=https://ollama.com/v1
export DGX_API_KEY=<key>

# point at a real LiteLLM proxy instead (🖥️ run it on the DGX):
export LITELLM_BASE_URL=http://<dgx>:4000
